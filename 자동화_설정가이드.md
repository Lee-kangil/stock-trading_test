# 자동화 설정 가이드 (Windows 작업 스케줄러)

매 영업일 자동으로 ① KIS 모의주문(듀얼모멘텀) ② 로컬 5전략 비교를 돌립니다.

## 구성
| 배치파일 | 하는 일 | 권장 시각 |
|----------|---------|-----------|
| `run_kis.bat` | KIS 모의계좌에 듀얼모멘텀 주문(`run_live.py`) | **평일 15:20** (장중이라 체결됨) |
| `run_paper.bat` | 로컬 5전략 가상비교(`multi_paper.py`) | **평일 16:00** (마감 후) |

실행 기록은 `data\run_kis.log`, `data\run_paper.log`에 쌓입니다.

---

## 1단계 — 먼저 수동 테스트 (중요)
자동 등록 전에, 배치가 잘 도는지 손으로 한 번 확인하세요. 파일 탐색기에서
`run_paper.bat`을 더블클릭 → 잠시 후 `data\run_paper.log`가 생기고 결과가 들어가면 성공.

KIS 쪽은 `.env`의 `KIS_LIVE=0`(시뮬레이션)인 상태로 `run_kis.bat`을 더블클릭해
`data\run_kis.log`에 `[DRY]` 주문이 찍히는지 확인하세요. (실제 주문은 충분히
확인한 뒤 `.env`에서 `KIS_LIVE=1`로 바꿉니다.)

## 2단계 — 작업 스케줄러에 등록
**방법 A) 명령어 한 줄씩 (간단)**
시작 메뉴에서 "명령 프롬프트"를 **관리자 권한**으로 열고 아래를 붙여넣기:

```cmd
schtasks /Create /TN "Stock_KIS" /TR "C:\Users\KangilLee\Claude\Projects\Stock_TickerFind\run_kis.bat" /SC WEEKLY /D MON,TUE,WED,THU,FRI /ST 15:20 /F

schtasks /Create /TN "Stock_Paper" /TR "C:\Users\KangilLee\Claude\Projects\Stock_TickerFind\run_paper.bat" /SC WEEKLY /D MON,TUE,WED,THU,FRI /ST 16:00 /F
```

**방법 B) GUI**
1. "작업 스케줄러" 실행 → 우측 "기본 작업 만들기"
2. 이름: Stock_KIS → 트리거: 매주, 월~금, 15:20
3. 동작: "프로그램 시작" → 프로그램에 `run_kis.bat` 전체경로 지정
4. 같은 방식으로 Stock_Paper(16:00, run_paper.bat) 하나 더 생성

## 3단계 — 등록 확인 / 삭제
```cmd
schtasks /Query /TN "Stock_KIS"          REM 등록 확인
schtasks /Run   /TN "Stock_KIS"          REM 지금 즉시 한번 실행(테스트)
schtasks /Delete /TN "Stock_KIS" /F      REM 삭제
```

---

## 주의사항
- **PC가 켜져 있어야** 그 시각에 실행됩니다. (절전/종료 상태면 안 돎)
  → 작업 속성 "조건" 탭에서 "작업을 실행하기 위해 절전 모드 해제" 체크 권장.
- **KIS 모의주문은 평일 장중에만 체결**됩니다. 그래서 15:20에 둡니다.
- 처음 몇 주는 `KIS_LIVE=0`(시뮬레이션)으로 두고 로그만 확인하다가, 신뢰가 쌓이면
  `KIS_LIVE=1`로 실제 모의주문을 보내세요.
- 모의계좌·로컬 모두 **6~8주 누적** 후 백테스트 기대치와 비교해 최종 판단합니다.
