## 2. 최종 전략 5개 (`src/strategies.py`)

## 4. 파일 구조 (`src/`)
- `strategies.py` : 5개 전략 + Strategy 정의
- `engine.py` : 범용 백테스트(`backtest_portfolio_generic`)·일일 스텝(`step_latest`), 레짐·보유기간 청산
- `backtest.py` : CostModel, 성과지표(CAGR/MDD/Sharpe/승률/손익비)
- `multi_backtest.py` : 5전략 비교 백테스트
- `multi_walkforward.py` : 연도별 OOS 검증
- `regime_backtest.py` : 레짐 필터 전/후 비교
- `multi_paper.py` : 로컬 5전략 페이퍼(각 2천만, 레짐 적용)
- `run_live.py` : KIS 모의계좌 듀얼모멘텀 1개(킬스위치·DRY_RUN·레짐)
- `kis_api.py` : KIS API(토큰·시세·잔고·주문·`get_balance_parsed`)
- `report.py` : HTML 리포트 생성
- (루트) `run_kis.bat`, `run_paper.bat` : 로컬 수동 실행용
- (루트) `.github/workflows/` : `kis_trade.yml`(15:05), `paper_report.yml`(16:05)

## 6. 자주 쓰는 작업 (다음에 꺼내 쓸 때)
- **로컬 수동 실행**: `.\run_paper.bat` (5전략), `.\run_kis.bat` (KIS). 또는 `python src\multi_paper.py`
- **리포트 직접 생성**: `python src\report.py` → `data\report_latest.html`
- **전략/금액 수정**: `src\strategies.py` 또는 설정값 고친 뒤 `git add . && git commit -m "..." && git push` → 다음 실행부터 반영
- **KIS 실거래(모의주문) 켜기/끄기**: GitHub Variables `KIS_LIVE` 0↔1
  - 1로 켤 때는 먼저 저장소 `data/live_state.json` 삭제(로컬·실제 동기화)
- **상태 리셋**: 저장소에서 `data/*_state.json` 삭제 → 다음 실행 때 새로 생성
- **자동 스케줄 시각 변경**: `.github/workflows/*.yml`의 cron(UTC) 수정

## 7. 보류/선택 항목
- **실잔고 연동**: 코드는 **로컬에 작성 완료**(`kis_api.get_balance_parsed`, `report.py` 실잔고 블록, `paper_report.yml`에 KIS env 추가). **GitHub 미반영** 상태.
  → 활성화하려면 `git push` 한 번이면 됨(그러면 리포트가 KIS 실제 잔고를 표시).
- 유니버스 확대(200→500/1000): `TOP_N` 값 변경. 단 느려지고 생존편향은 안 풀림.
