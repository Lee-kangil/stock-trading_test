# 6주차 — 실거래 API 연동 가이드 (한국투자증권 KIS)

> 목표: 신호를 브로커 API로 보내 자동 주문까지 잇는다.
> **단, 반드시 모의투자(paper)부터. 실거래·송금은 본인 승인 하에만.**

---

## 안전 3원칙 (가장 중요)
1. **모의투자 먼저** — `.env`의 `KIS_ENV=paper`. 실전(real)은 모의에서 수 주 검증 후에만.
2. **DRY_RUN 기본** — `KIS_LIVE=0`이면 주문을 *계산만 하고 전송하지 않습니다.* 충분히
   확인한 뒤에만 `KIS_LIVE=1`로 실제 전송(그래도 paper면 모의계좌로만 나감).
3. **킬스위치** — 누적손실이 −15%를 넘으면 신규 매수를 자동 차단(청산만 허용).

---

## Step 1. API 신청 & 키 발급
1. 한국투자증권 계좌 개설 + ID 연결
2. [KIS Developers](https://apiportal.koreainvestment.com/) 에서 Open API 서비스 신청
3. **앱키(App Key) / 앱시크릿(App Secret)** 발급 — ⚠️ 모의투자용과 실전용이 **별도**
4. 모의투자 계좌번호 확인 (앞 8자리 / 뒤 2자리, 종합은 01)

## Step 2. 패키지 설치
```powershell
pip install -r requirements.txt   # requests, python-dotenv 추가됨
```

## Step 3. 키를 .env에 설정
```powershell
copy .env.example .env
```
그런 다음 `.env`를 열어 앱키·앱시크릿·계좌번호를 채웁니다.
`.env`는 `.gitignore`로 깃에 올라가지 않습니다(키 유출 방지). **키를 코드에 직접 쓰지 마세요.**

## Step 4. 연결 확인 (주문 없이)
```powershell
python src\run_live.py
```
- 처음엔 `KIS_LIVE=0`(시뮬레이션)이라 **실제 주문이 나가지 않습니다.**
- 출력에서 ENV=paper, 키/계좌 설정 여부, 오늘의 매매 의도([DRY])를 확인합니다.

## Step 5. 모의투자 실제 주문 (신중히)
충분히 확인했다면 `.env`에서 `KIS_LIVE=1`로 바꾸고 다시 실행하면,
**모의투자 계좌로** 실제 주문이 전송됩니다([SENT]). 실전 계좌가 아닙니다.

---

## 파일 구성
| 파일 | 역할 |
|------|------|
| `src/kis_api.py` | KIS 클라이언트 (토큰·시세·잔고·주문). 모의=VTTC, 실전=TTTC tr_id 분기 |
| `src/run_live.py` | 신호 → 주문 브리지. DRY_RUN·킬스위치 내장 |
| `.env` | 키/계좌/스위치 (깃 제외) |
| `data/kis_token.json` | 토큰 캐시 (1분당 1회 발급 제한 회피) |

---

## 알아둘 점
- 토큰은 발급이 **1분당 1회**로 제한됩니다 → 캐시를 사용합니다(자동).
- 모의투자 계좌는 REST 호출 제한이 낮습니다(연속 호출 많으면 `EGW00201`).
- 엔드포인트·tr_id는 KIS 명세 기준이며 공지 없이 바뀔 수 있습니다. 이상 시
  공식 샘플로 대조하세요: https://github.com/koreainvestment/open-trading-api
- 종가 신호 → 실제는 보통 '다음 날 시가' 체결이라 약간의 괴리가 생깁니다.
  실전 전환 전, 모의에서 이 괴리를 충분히 관찰하세요.

> 면책: 본 코드는 참고용입니다. 실거래 주문·손익의 책임은 전적으로 본인에게 있습니다.
> 저는 주문을 대신 실행하거나 자금을 이동하지 않습니다.
